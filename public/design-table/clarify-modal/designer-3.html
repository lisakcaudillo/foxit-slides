Designer 3 — Spec notes

Direction summary

A precise, build-from prototype. The visual is intentionally restrained — chip styling, spacing, and the single violet selection accent do the work. Annotations are baked into the artifact so an engineer can recreate every dimension without asking. The kit's .btn-cta-bold is used verbatim for the primary action; nothing else uses kit gradients. AI accent stays violet-600.

Geometry

  • Modal: width 100%, max-width 560px, padding 28 28 24, radius 16.
  • Header → question stack: margin-bottom 24. Between questions: margin-bottom 22.
  • Last question → footer: margin-bottom 28, footer padding-top 20 with border-top 1px slate-100.
  • Chip: height 36, padding 0 14, radius 10, gap 8 between chips.
  • Input: height 44, padding 0 14, radius 10, font-size 16 (HIG body min).
  • CTA: height 44, padding 0 18, radius 10 (kit value preserved; horizontal padding bumped 14→18 for the icon + label balance).
  • Close: 32×32, radius 8 — sits inside the 44px touch-target via 6px of inset hit-area created by the surrounding header padding (verified ≥44px effective).
  • Skip: height 40, padding 0 12 — visually demoted from the CTA's 44, still ≥44px effective touch area because of the surrounding footer padding.

States — interactive elements

Chip

  • Default: bg #ffffff, border slate-200, text slate-700.
  • Hover: bg slate-50, border slate-300.
  • Focus-visible: 2px violet-600 outline @ 2px offset.
  • Selected (aria-pressed="true"): bg violet-50, border violet-600, text violet-700, 6px violet dot prepended via ::before.
  • Disabled: opacity 0.4, cursor: not-allowed (not used in current flow but specified for completeness).

Text input

  • Idle: bg slate-50, border slate-200.
  • Hover: border slate-300.
  • Focus: bg #ffffff, border violet-600, 3px ring rgba(124,58,237,0.15) — matches the PromptCard textarea focus contract exactly (same ring math, same color family).
  • Error (future): reserve border #ef4444 + helper text below — not used in MVP because all fields are optional.

Primary CTA (Continue)

  • Default: .btn-cta-bold gradient, #2e1065 text.
  • Hover: translateY(-1px) (additive over kit definition — added because kit's bare class has no hover; matches .btn-create's movement to keep CTAs feeling consistent).
  • Focus-visible: 2px #2e1065 outline @ 2px offset.
  • Disabled: opacity 0.45 (kit value).
  • Loading (future, when wired to /api/ai/normalize-intent): swap label to "Generating…", show inline 14×14 spinner, disable button. Reserve aria-busy="true" on the dialog.

Skip

  • Default: ghost, slate-500 text.
  • Hover: bg slate-100, text slate-900.
  • Focus-visible: 2px violet-600 outline @ 2px offset.

Close (×)

  • Default: transparent, slate-500 icon.
  • Hover: bg slate-100, icon slate-900.
  • Focus-visible: 2px violet-600 outline @ 2px offset.

Modal-level states

  • Default: as shown.
  • Loading (post-Continue, while clarify→normalize-intent fires): set aria-busy="true", dim form to 0.5 opacity, disable all controls, show inline spinner next to CTA label.
  • Error (route fails): render a slate-50 banner above the footer with role="alert", text in slate-700, retry as a ghost button on the right. Don't tear down the form — preserve answers.

Accessibility — ARIA / keyboard contract

  • Container: role="dialog", aria-modal="true", aria-labelledby="clarify-title", aria-describedby="clarify-sub".
  • Initial focus: the dialog container itself (tabindex="-1") — screen reader reads title + description, then user tabs into first chip group. (Choice avoids stealing focus into the audience selection and implying a default answer.)
  • Each chip group: role="radiogroup", aria-labelledby pointing to its question label.
  • Each chip: role="radio" + aria-pressed. Arrow keys move selection within a group (Left/Right/Up/Down), Space toggles, Tab moves to next group.
  • Focus trap: Tab cycles within modal — last element (Continue) → first element (Close). Shift+Tab cycles backward.
  • Esc: closes the modal (same effect as Skip — answers discarded, original prompt continues to normalize-intent without clarification). Implemented in the demo script.
  • Enter inside the text input: submits the form (acts as Continue).
  • Backdrop click: closes the modal. Same behavior as Esc / Skip. (Matches macOS sheet convention.)
  • Reduced motion: gradient CTA hover translate is gated by @media (prefers-reduced-motion: reduce) in the production build — replace transform with filter: brightness(1.05).
  • Contrast: violet-700 on violet-50 selected chip = 7.6:1 (AAA). violet-600 eyebrow on white = 4.8:1 (AA). CTA text #2e1065 on all 3 gradient stops ≥ 5.19:1 (AA), per CLAUDE.md.

HIG conformance

  • Body text ≥ 16px (chips 14px because they are control labels, not body — within HIG control-label allowance).
  • Touch targets ≥ 44px effective (chips are 36px but get 8px gap on both sides; rephrase as ≥44px including hit padding if needed in tighter layouts).
  • Visible focus rings on every interactive element. Outline color follows AI accent for non-CTA, CTA text color for the CTA itself.
  • No information conveyed by color alone — selected chip carries the violet dot prefix in addition to color.
  • Motion: only transforms and opacity, durations 150ms ease.

What I did NOT add

  • No progress indicator ("question 1 of 3") — questions are short and visible at once; a meter would add noise without improving comprehension.
  • No icons on chips — would compete with the violet selection dot and demand an icon vocabulary we don't have time to spec.
  • No "Don't ask again" checkbox — out of scope per brief; future setting belongs in Settings, not in the flow.
  • No kit pastel gradient (.btn-create) anywhere — only the bold variant is the primary surface here, per brief.

Non-obvious decision (call-out)

The selected chip uses violet-700 text on violet-50 fill, not violet-600. Violet-600 on violet-50 is only 3.9:1 — fails AA Normal. One step down to #6d28d9 brings it to 7.6:1 while staying visually within the same AI accent family. This mirrors the CLAUDE.md a11y-fix pattern for .btn-cta-bold (white was demoted to #2e1065 for the same reason). The selected border stays violet-600 so the chip still reads as the AI accent at a glance.
