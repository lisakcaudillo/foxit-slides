# Manager — Synthesis: Clarify as a 3-Lens Detector

## Core finding from the three strategy docs

The three strategies are **not mutually exclusive lenses competing for the same slot.** They are three independent **detectors** that should ALL run on every prompt and only surface the question(s) they fire on. Both Strategy 2 (data) and Strategy 3 (narrative) explicitly defer to each other when their lane doesn't apply. Strategy 1's empty-state-as-success-state behavior is the right default for everyone.

The Clarify route's job is to run the three detectors, dedupe overlaps, cap total questions at 2–3, and surface only what fired.

## The three lenses

| Lens | What it detects | Priority | Why |
|---|---|---|---|
| **Disambiguation** | Genuinely ambiguous nouns ("Mercury", "Atlas", a name that's a person AND a product) | **Highest** | Wrong topic = wrong deck. Cheapest to ask (one chip pick), highest cost to skip. |
| **Data-grounding** | Quantitative claims with no numbers ("beat the target", "doubled", "tripled", "exceeded") | **High** | FR11 forbids hallucinated metrics. Without grounding, charts become fakes or placeholders. |
| **Topic-narrative** | Story-shaping decisions only the human knows ("what drove the beat", "what to soften") | **Medium-High** | Without it, the deck is a tidy template fill — structurally correct, emotionally generic. |

## Priority rules

1. **Disambiguation always wins.** If it fires, it's question #1, no exceptions.
2. **Data + Narrative are co-equal in the remaining slots.** If both fire, take the single highest-leverage question from each. Don't surface a second data question when a narrative question fired (or vice versa) — that's how the modal becomes a form.
3. **Cap: 2 questions for organic prompts, 3 for Inspire-Me prompts** (which are thin by definition).
4. **Empty-state is success-state.** If zero lenses fire, return `{ questions: [] }` and the modal does not open. No confirmation theater. Generation proceeds.

## Anchor prompt — worked example
> "Make a deck about our Q1 sales — we beat the target"

| Lens | Fires? | What it would ask |
|---|---|---|
| Disambiguation | No | — |
| Data-grounding | **Yes** | "Got the actual numbers? Paste them however you have them — otherwise I'll use placeholders you can fill in later." *(text, multiline, skippable, hint: `Target: $2M  Actual: $2.4M`)* |
| Topic-narrative | **Yes** | "What made the difference this quarter?" *(select: `New product` / `Renewal cycle` / `Big single deal` / `Team execution` / `Market tailwind` / `Something else…`)* |

**Final Clarify output: 2 questions.** Narrative first (shapes the blueprint), data second (grounds the content). The user can answer one, both, or neither.

> **Microcopy note:** all question strings in this doc are samples — the live ones are generated by `/api/ai/clarify` per-prompt. The route's system prompt must enforce conversational, jargon-free phrasing. Bad: "What drove the beat?" / "Define your success metric." Good: "What made the difference this quarter?" / "What's the story behind the numbers?"

## Second prompt — only one lens fires
> "Make a deck on agentic AI"

| Lens | Fires? | What it would ask |
|---|---|---|
| Disambiguation | Marginal — "agentic AI" is broad but not ambiguous in the planet-vs-band sense | (skip) |
| Data-grounding | No quantitative claim | (skip) |
| Topic-narrative | **Yes** | "What's your angle — capabilities, market landscape, risks, or how to build with them?" *(select with text fallback)* |

**Final Clarify output: 1 question.** Modal is shorter. Visible Skip button still present.

## Third prompt — zero lenses fire
> "Onboarding deck for new sales hires — sections on tools, ICP, comp plan, key contacts, first-week goals"

User already supplied audience, structure, and content scope. Clarify returns `{ questions: [] }`. **Modal does not open.** Generation proceeds straight to blueprint. The visible feedback is the blueprint appearing, not a "Sounds good!" modal.

## Implementation map → `/api/ai/clarify`

The route runs one Claude call with three structured detection tools (`detect_disambiguation`, `detect_data_gap`, `detect_narrative_gap`). Each tool returns `{ fired: bool, question: ClarifyQuestion | null, leverage: 0–1 }`. The orchestrator:

1. Collects all `fired: true` results
2. Sorts by lens priority (disambig > data + narrative tied → break by `leverage` score from Claude)
3. Caps at 2 (organic) or 3 (Inspire-Me)
4. Returns `{ questions }` — possibly empty

The modal opens only if `questions.length > 0`. Skip on the modal sends `{ clarifications: {} }` to normalize-intent, which proceeds as if Clarify never fired (no penalty). Answers are appended as a `clarifications` object that the blueprint stage prioritizes over the original prompt when sections conflict.

## What this gives up vs each pure strategy

- vs Strategy 1 (Minimal): one more question fires on average per prompt. Justified because Strategy 1 leaves narrative on the table, and narrative is what makes the output feel authored.
- vs Strategy 2 (Data-max): only ONE data question per prompt, not three. Justified because over-asking on data when the user just wanted a quick deck is the exact friction Lisa flagged.
- vs Strategy 3 (Narrative-max): only ONE narrative question per prompt. Justified because if data is also missing, the user needs to know that's askable too — combining the lenses respects both lanes.

## The single most important rule

**Clarify only asks what only the human can answer AND what materially changes the deck.** Anything else — audience, tone, detail, generic preferences — is a setting. Settings live in the Customize popover, not in Clarify.
