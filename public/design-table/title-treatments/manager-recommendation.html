Manager Recommendation — Title Treatments

This is my synthesized fourth option — the strongest family of six title treatments, taking the typographic grammars from Designer 1 that genuinely read as different page layouts, the deterministic theme→treatment selector and overridable swap from Designer 2, and token-traceable anatomy throughout. The editorial Anchor Lisa liked is the flagship; the other five each wear a layout move no body slide ever wears, so a browse across themes reads as variety, not one template recolored.

Recommendation summary

The six that make the cut

  • 1Anchor — eyebrow + hairline rule + serif italic accent word + byline strip. The flagship; the bookish page Lisa anchored on.
  • 2Drop-Cap Opening — oversized serif initial sets the headline like a novel's first page. Unmistakably a cover.
  • 3Chapter Divider — big numeral + "Chapter Three" kicker + hairline divider. A book's section opener.
  • 4Statement — the quote/claim IS the headline, baseline-anchored, one italic emphasis. Confident, chrome-light.
  • 5Folio Masthead — title rail top (masthead + issue no.) + display headline + byline strip — page bracketed top & bottom.
  • 6Spotlight — title floats over a full-bleed motif glow + soft scrim. For dramatic dark / branded themes.

What I'd cut, and why

All-Caps Masthead (D1 #2) — merged into Folio Masthead. The caps-banner and the magazine folio were two takes on the same "newspaper framing" idea; the folio is richer (top rail + issue line) and absorbs the caps banner as a styling variant. One masthead, not two.

Stacked Baseline (D1 #4) — cut. Word-per-line is a beautiful effect but it's a variant of Statement (oversized title, one italic turn), not a distinct grammar. Folding it in keeps the family from having two "big title" entries that blur together.

Centered Colophon (D1 #6) — cut. Its symmetric, centered, rules-above-and-below structure is the same skeleton as Chapter Divider minus the numeral. Chapter Divider is the stronger, more legible version of "centered & calm."

Split Panel (D2) — deferred, not in this family. A type/motif split is excellent but overlaps the existing split cover tier in cover-tiers.ts; titles should be type-led so they never render as an empty image box. Keep it as a cover-tier concern, not a title treatment.

The one-rule mapping — automatic variety, always overridable

Variety is not authored 42 times. A single deterministic selector — titleTreatmentForTheme(theme) — reads three signals already on every Theme object (archetype, category, tone) and returns one of the six. It mirrors the exact shape of coverTierForTheme() in cover-tiers.ts today: a small per-theme override map first, then a data-driven derivation. Evaluated top-to-bottom, first match wins:

  1. Named override? A short per-theme map wins outright (the same escape hatch as THEME_COVER_TIER). For editorial judgment on the few themes whose best treatment isn't obvious from signals.
  2. archetype = editorial → legal theme → Anchor (the bookish flagship); otherwise → Folio Masthead (editorial business reads as a banner).
  3. archetype = cinematic → Spotlight (dark / branded drama → full-bleed motif glow + scrim).
  4. archetype = warm → Chapter Divider (centered, breathing, friendly).
  5. archetype = product → Statement (bold creative covers; the title fills the frame).
  6. Fallback → Anchor — type-led, never an empty image box, never breaks.

Drop-Cap is the editorial override slot: ledger → Drop-Cap, so finance/annual-report themes get the "novel's first page" feel instead of a second banner. Because archetype drives the primary branch, the four archetypes guarantee at least four distinct treatments before overrides — and overrides push the look-alikes apart.

The recommended six — each on a well-fit theme

Real pageBg, fonts, title colors, and the theme's own CoverArt motif behind the words at 12–22%. The treatment under each is the one the rule picks for that theme. Every headline and label clears WCAG AA.

Override the auto-pick — "Try a different title"

Design intelligence picks a sensible default; the user always decides. Right-click any cover thumbnail to open the same context menu that already hosts Try a different cover and Try a different layout — now with a Try a different title group listing all six. The current pick is checked; choosing another re-renders the cover in place, keeping the theme's colors, fonts, and motif. One gesture, plain-language labels, instant preview (pure re-layout of the same tokens — no regeneration, no network call).

Why this interaction shape

  • Mirrors what exists. The same right-click menu already carries Try a different cover and Try a different layout — users learn one gesture, not three.
  • Suggested, not forced. The auto-pick shows checked; "Reset to suggested" returns to the system's choice after experimenting (overridable design intelligence).
  • Cheap to preview. Switching is a pure re-layout of the same theme tokens — instant on click, no regeneration.
  • Plain language. The menu says "title" and the treatment's plain name (Anchor, Drop-Cap), never titleStyle / "tier" / "archetype". Internal names never reach the surface (the OCR → "Grab Text" precedent).
  • Consistent with P-UX2. The picker appears only on intent (right-click) — covers stay calm by default.

How this builds on the design Lisa liked

The flagship Anchor is the exact treatment Lisa anchored on — the hardcover-book title page: an eyebrow with a hairline rule, a large headline with one italic accent word, the theme's CoverArt motif promoted to ambient texture at 12–22%, and a bottom byline strip that frames the page as a cover rather than a body slide with bigger text. Nothing about it changes here; it's slot one and the guaranteed fallback.

The other five extend that same DNA into different page grammars — moves no body slide wears — so the family reads as variety, not one template recolored. Drop-Cap borrows the book's first page; Chapter Divider borrows its section opener; Statement and Spotlight borrow the cover's confidence (one fills the frame, one floats over a motif glow); Folio Masthead borrows the magazine's top rail. Each keeps the editorial signatures — eyebrow, hairline, italic accent, byline — so they're recognizably siblings.

Crucially, the variety is automatic and buildable, not 42 bespoke pages. A single deterministic selector keyed on (archetype, category, tone) assigns each theme its default treatment, mirroring the exact shape of coverTierForTheme() already shipping in cover-tiers.ts (override map first, then data-driven derivation). And every pick is overridable through the same "Try a different…" right-click users already know — so the system suggests, and Lisa's users decide.
