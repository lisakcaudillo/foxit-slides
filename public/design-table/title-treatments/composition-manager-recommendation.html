Design Table · Round 2 · Manager Recommendation

Manager Recommendation — Dynamic Image + Title Composition

The title slide is the editorial title kit Lisa liked in round 1 — eyebrow → hairline rule → serif headline (one italic accent word) → byline — now mounted on a real generated image every time. A small deterministic engine, shaped exactly like coverTierForTheme(), reads the image and the theme and decides which of six composition forms to use, anchors the title opposite the image, and adds a brightness-tuned scrim only where text overlaps the image. Image and title co-exist by construction; across a deck the picks vary so no two covers feel stamped.

1 · Recommendation

Six composition forms — the union of Designer 1's craft set and Designer 2's engine vocabulary, deduped down to the forms that each earn their place. Type-only stays as the guaranteed fallback (it's the only form with no image, so it lives in the engine but never appears in the image gallery below).

Recommended set — six forms

1Vertical half
Image one half, title column the other. Mirrors L↔R across the deck via a rotation counter. The corporate workhorse — zero overlap, crisp type.
2Diagonal split
Clip-path slash; title in the complementary triangle (always opposite the image). The energetic variety form for creative / product themes.
3Band — image top
Image in a top band, title anchored on the clean theme band below. Best for square-ish or busy images that crop badly into a half.
4Band — image bottom
Image below, title on clean theme bg above. The bright-image safety valve — when an overlay would lose contrast, band it.
5Full-bleed + scrim
Image fills the frame; title overlaid on a brightness-tuned scrim. The dramatic form for cinematic / dark themes.
6Type-only (fallback)
No image. Theme bg + pattern + corner CoverArt motif + gradient title. The never-breaks baseline = type tier today.

What I'd cut / merge

  • Merge Designer 1's "vertical half" + its "mirror" into one form with an imageSide param. They're not two forms — they're one form the rotation counter flips. Same for the two diagonal mirrors.
  • Cut a distinct "full-bleed side-scrim" vs "full-bleed bottom-scrim" as separate forms — it's one full-bleed form; scrim geometry is computed from where the title lands, not authored.
  • Keep both bands (top and bottom). They look symmetric but fire on opposite signals — bottom-band is the only contrast-safe answer for a bright image, so it earns its slot.

The one selection rule (engine, not layout)

  • Same deterministic shape as coverTierForTheme(): a small explicit map for named themes, then a derive-from-identity fallback.
  • Inputs → one output: image present? · aspect · brightness · theme category/tone/archetype · headline length → form + title position (always opposite the image) + scrim.
  • Overridable: "Try a different title layout" swap re-picks the form; the engine still guarantees opposite-position + legibility on any manual choice.

2 · The one rule the deck engine runs

Rules evaluated top-to-bottom, first match wins — exactly like the explicit map + deriveCoverTier() fallback in cover-tiers.ts. titlePosition = opposite(imageSide) is computed, never authored. A scrim exists only for full-bleed; its alpha is tuned to image brightness so both dark and bright images clear WCAG AA.

Scrim alpha for full-bleed = clamp(0.35 + (0.5 − imageBrightness)·0.7, 0.40, 0.78) → darker images get a lighter scrim, bright images a heavier one; both land ≥ 4.5:1. A per-deck rotation counter breaks ties between equally-valid forms so a 6-slide deck never repeats a form back-to-back.

3 · The forms, on real themes with real images

Five image-bearing forms (the sixth is type-only, shown small at the end as the fallback). Each is a 16:9 cover with a real generated image and the editorial title kit adapting to the open region.

4 · How this builds on what Lisa liked + fixes the round-1 gap

Builds on the design Lisa liked

The editorial title kit is untouched. Eyebrow → hairline rule → serif headline with one italic accent → byline is exactly the round-1 treatment Lisa approved. It carries straight through every composition here.

The deterministic engine is the shape she trusts. The selection rule mirrors coverTierForTheme() — explicit per-theme map first, identity-derived fallback second — so it's the same predictable, overridable system already shipping for cover tiers, just with a richer vocabulary (six forms instead of three tiers).

Fixes the round-1 gap

Round 1 had no images — this makes image + title co-exist. Every form here mounts the editorial kit on a real generated image, and the title re-anchors to the open region opposite the image so the two never fight. Where they must overlap (full-bleed), a brightness-tuned scrim guarantees legibility.

Dynamic, not a fixed cover. The deck engine generates one image and places it as a half, a diagonal cut, a band, or full-bleed depending on the image and theme — and the title treatment adapts to match. Type-only remains the honest fallback when no image exists, never an empty box.
