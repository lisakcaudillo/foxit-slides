Dynamic Title Composition Engine

The title slide is not a fixed layout — it is the output of a decision. The deck orchestrator already generates a cover image; this engine reads four signals (is an image available, its orientation & brightness, the theme's tone/archetype, and the headline length) and chooses a composition form + title position + scrim so the image and the title always co-exist instead of fighting. Across a deck the picks vary, so no two title slides feel stamped from the same mold — and when no image is available it falls back to a clean type-only cover, never an empty image box.

1 · The decision the engine makes

Inputs in, one composition out. The title position is always chosen opposite the image, and a scrim is added only when the title overlaps the image — its opacity tuned to the image's brightness so text clears WCAG AA.

IN image available? IN image orientation (portrait / landscape / square) IN image brightness (dark / mid / light) IN theme archetype (cinematic / editorial / warm / product) + tone IN headline length (short / long)

Rules are evaluated top-to-bottom; the first match wins. A per-deck rotation counter breaks ties between equally-valid forms so a 6-slide deck doesn't pick the same form twice in a row.

When the inputs say…Composition formTitle positionScrimWhy it co-exists
No image available, or generation failed type-only left-column none Guaranteed fallback. Theme bg + pattern + corner motif carry it. Never an empty frame.
Portrait image (tall) · any tone vertical-half (image side) opposite column edge feather only A tall image fills a half-panel cleanly; title gets the other half — zero overlap.
Landscape image · cinematic / dark theme · short headline full-bleed-overlay bottom (or centered) bottom gradient, brightness-tuned Drama wants the whole frame; a bottom scrim guarantees the title reads over any image.
Landscape image · editorial / product theme · long headline vertical-half opposite column none (no overlap) Crisp type matters for corporate decks; image stays in its lane, words in theirs.
Square image · warm / editorial theme band-image-top bottom band none (banded) A square crops awkwardly into a half; a horizontal band frames it and seats the title below.
Landscape · creative / product · variety slot diagonal-split right, against the slash none (clipped) Energetic alternative used for variety; the clip keeps image and title on separate sides.
Bright / busy landscape · title must overlap band-image-bottom top band none (banded) When an image is too bright for an overlay, banding it below keeps the title on clean bg.

Position is computed, never authored: titlePosition = opposite(imageSide). Scrim only exists for full-bleed-overlay; its alpha = clamp(0.35 + (imageBrightness − 0.5)·0.6, 0.4, 0.75) so darker images get a lighter scrim and bright images get a heavier one — both land ≥ 4.5:1.

2 · Eight engine outputs (real images, real themes)

Same engine, different inputs → eight distinct title slides. Each label names the form, the chosen title position, the theme, and the rule that fired. Headline emphasis across all forms is the approved library style — one word set italic in a contrasting accent color drawn from the theme — not a whole-title gradient.

3 · "Title layout" — multi-select & overridable

The auto-pick is a starting point, not a verdict — and it's a multi-select. Tick several layouts to define the set the engine is allowed to rotate through across the deck (so variety stays on-brand), or override a single slide outright. Right-click the title thumbnail → Title layout. Hover the menu below to preview (CSS-only mock).

Slide 1 · Title

Right-click → Title layout · or use the cover toolbar's “Try a different title layout”. Checked = enabled for this deck.

Every form reuses the same scrim + opposite-position logic, so any pick still reads well — the user chooses the set of forms, the engine still guarantees legibility and rotates among the enabled ones for variety. Tick a single layout to force every cover to it; tick several to let the deck vary within your chosen set; pick Type only to drop imagery entirely. Switching back to “Auto” re-runs the picker.
